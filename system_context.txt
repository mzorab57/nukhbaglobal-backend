NUKHBAGLOBAL
ڕێنمایی و پێکهاتەی گشتی داتابەیس و شێوازی کارکردنی سیستەم

ئامانج:
ئەم فایلە بۆ هەڵگرتنی تێگەیشتنێکی گشتی لە business logic و بنەمای داتابەیسەکە دروست کراوە، بۆ ئەوەی هەر کارێک لە backend لەسەر بنەمای یەک تێڕوانین بکرێت.

کورتەی business:
ئەم سیستەمە بۆ بەڕێوەبردنی گەشت و فرۆشتنی بلیت دروست کراوە.
پشتیوانی دەکات لە:
- پارەدانی ئۆنلاین و دەروازەکانی وەک FIB, FastPay, ZainCash, QiCard
- کڕینی بلیت بە کۆمەڵ
- دەرکردنی بلیت بە QR Code
- سکانکردنی بلیت لە دەرگا/پاس
- چاودێری خەرجی و دارایی
- ئەرشیفکردنی گەشتە کۆنەکان
- لۆگکردنی کردارەکانی ستاف

--------------------------------------------------
1. users
--------------------------------------------------
ئەرک:
پاشەکەوتکردنی زانیاری کارمەندان بۆ چوونەژوورەوە و بەڕێوەبردنی سیستەم.

ستونەکان:
- id: ناسنامەی ناوازە و ئۆتۆماتیکی بەکارهێنەر
- name: ناوی تەواوی کارمەند
- email: ئیمەیڵی ناوازە بۆ login
- password: وشەی نهێنی hashed
- role: ڕۆڵی کارمەند، وەک admin / scanner / accountant
- permissions: دەسەڵاتی ورد بە شێوازی JSON
- status: 1 چالاک، 0 ڕاگیراو
- deleted_at: soft delete
- created_at
- updated_at

--------------------------------------------------
Business Flow ی گشتی سیستەم
--------------------------------------------------
1. admin یان ستاف country و city زیاد دەکات
2. event سەرەکی دروست دەکرێت
3. بۆ event ـەکە sub_event/itinerary زیاد دەکرێت
4. جۆرەکانی ticket بۆ event یان sub_event دادەنرێن
5. customer order دروست دەکات و ticket ـەکانی هەڵدەبژێرێت
6. order_items ژمارە و نرخەکانی هەر ticket جیا دەکاتەوە
7. payment record دروست دەبێت
8. ئەگەر payment سەرکەوتوو بوو، event_tickets دروست دەبن
9. لە دەرگا scanner بلیت بە QR Code سکان دەکات
10. status ـی بلیت دەگۆڕدرێت و activity_logs تۆمار دەکرێت
11. expenses بۆ هەر event زیاد دەکرێن بۆ حسابات
12. دوای تەواوبوونی event، دەتوانرێت بۆ past_events ئەرشیف بکرێت

--------------------------------------------------
جیاوازییەکانی نێوان وەسفە business ـەکە و schema ـی ئێستا
--------------------------------------------------
1. gateway list وەک FIB, FastPay, ZainCash, QiCard لە business ـەکە دیارە، بەڵام لە schema ـدا `gateway_name` وەک VARCHAR دانراوە بۆ ئەوەی flexible بێت
2. ticket_code پێویستە بە شێوازی UUID/QR logic بەرهەمبهێندرێت لە logic ی backend، نەک تەنها لە داتابەیس
3. state management ی `reserved_count`, `sold_count`, order expiration, webhook verification هەموویان business logic ـن و دەبێت لە backend چارەسەر بکرێن

--------------------------------------------------
گرنگ بۆ هەنگاوەکانی داهاتوو
--------------------------------------------------
- ئەم فایلە reference ـە، نەک migration
- schema ـی SQL لە فایلێکی جیا هەیە: `nukhbaglobal_schema.sql`
- پێش دروستکردنی logic، باشە business rules بخرێنە service layer
- پێویستە دواتر بڕیار بدرێت backend framework چی بێت، بۆ نموونە Laravel یان Node.js
- ئەگەر بەپێی ئەم وەسفە کار بکەین، پێویستە schema ـی ئێستا لە هەندێک شوێندا نوێ بکرێتەوە
